# NovaPlan Backend Pytest Suite

These tests are testing assets only. Hugging Face startup runs them only when `APP_ENV=TEST`; `APP_ENV` unset or `APP_ENV=PROD` skips tests and starts NovaPlan normally.

## Created Files

| File | Purpose |
|---|---|
| `requirements-test.txt` | Test-only Python dependencies |
| `pytest.ini` | Pytest configuration and import paths |
| `tests/conftest.py` | Shared dummy dispatcher/tracker fixtures and safe test environment defaults |
| `tests/test_rasa_actions.py` | Unit tests for Rasa custom action behavior, events, dispatcher messages, custom payloads, DB error handling, handover, and fallback |
| `tests/test_rasa_action_helpers.py` | Unit tests for slot helpers, parser helpers, formatter helpers, mapping helpers, and transport builder |
| `tests/test_rasa_services.py` | Unit tests for Travelpayouts, Climatiq, Geoapify/OSM hotel, Wikipedia, RestCountries, and offset service behavior with mocked external calls |
| `scripts/run_tests.sh` | Optional Docker startup test gate controlled by `APP_ENV` |

## Commands

Run from `backend`:

```bash
APP_ENV=TEST
pip install -r requirements-test.txt
pytest -v
pytest --cov=rasa.actions --cov=api --cov-report=term-missing
pytest --cov=rasa.actions --cov=api --cov-report=html
pytest -v > pytest_results.txt
```

The Rasa action server imports custom action modules through the `rasa/actions` path. Local coverage commands may report `rasa.actions` as not imported even when action tests execute successfully. Treat `pytest -v` and the action/service test files as the primary unit-test evidence; coverage output is supplementary.

In the Hugging Face Docker image, startup tests are gated by `APP_ENV`.

| `APP_ENV` | Startup behavior |
|---|---|
| unset or `PROD` | Skip tests and start NovaPlan normally |
| `TEST` | Run pytest and Rasa test commands before Supervisor starts services |

## Mocking Policy

The suite avoids live external dependencies:

| Dependency | Test Strategy |
|---|---|
| Travelpayouts | Token disabled or `_fetch_travelpayouts_flights` monkeypatched |
| Climatiq | API key disabled, zero-carbon shortcut tested, payload validation tested |
| Geoapify | API key disabled or `_fetch_geoapify_hotels` monkeypatched |
| Wikipedia REST API | `utils.http_client.get` dependency monkeypatched through service module |
| RestCountries | `utils.http_client.get` dependency monkeypatched through service module |
| Neon PostgreSQL | Persistence helpers and itinerary service calls monkeypatched |

## Evidence

```bash
pytest -v > pytest_results.txt
```

Runtime evidence is written inside the container under `/app/results` when `APP_ENV=TEST`. Rasa also writes native outputs under `/app/rasa/results`, and the test script copies those outputs into `/app/results/rasa`.

Runtime-generated files include:

| File | Purpose |
|---|---|
| `/app/results/evidence_report.html` | Structured HTML report with dataframe-style evidence table |
| `/app/results/evidence.json` | Structured machine-readable evidence summary |
| `/app/results/evidence_tables.json` | JSON table rows for dataframe/report use |
| `/app/results/backend_validation_report.md` | Markdown evidence report generated by the Hugging Face test run |
| `/app/results/test_evidence_summary.csv` | CSV dataframe-style evidence summary |
| `/app/results/test_summary.txt` | Plain text structured status summary |
| `/app/results/pytest_results.txt` | Raw pytest command output, supporting artifact only |
| `/app/results/rasa/nlu/intent_report.json` | Rasa NLU classification report |
| `/app/results/rasa/nlu/intent_errors.json` | Rasa NLU incorrect predictions |
| `/app/results/rasa/core/story_report.json` | Rasa Core story report |
| `/app/results/rasa/e2e/rasa_e2e_results.txt` | Raw Rasa E2E command output, supporting artifact only |
